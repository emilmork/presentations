REST API Design
  • URI Format
    • URI = scheme "://" authority "/" path [ "?" quary] [ "#" fragment]
  • Don´t use slash (/) in the end an URI
    • http://api.path.org/resource/
  • Use hyphens (-) for readability
    • http://api.path.org/this-is-readable
  • Don`t use underscores (_)
    • Depending on browsers, editors, "_" can get weird or hidden.
  • Use lowercase letters.
    • http://API.PATH.org
REST API Design - Resource Modeling
  • Collection
    • ...org/emplyees
  • Document
    • /employees/emil
    • /employees/emil/hobbies/yodeling
  • Stores
    • PUT /employees/1234/hobbies/banjo
  • Controller
    • POST /alerts/1234/resend
  • Document, Store - Substantiv
  • Collection names - Substantiv flertall
  • Controller names - Verb
